Manage projects

01.10.2018

This document describes how creating, editing and deleting projects is managed.
It's rather cryptic and probably most useful for myself.

Inspired by PouchDB's docs.

Symbols:

  • : done
  • : written but untested
  • : todo

create

  1. app-side:

    • Tree includes node "Projekte: eigene" (besides "Projekte: mitarbeit")

      • top node is empty text-input that creates new project when user inputs name
    • user names new project
    • user confirms the new name
    • app creates projectDef doc in message db
    • create project db
    • create project collection
    • create projectDef doc in project collection
    • sync project collection
    • show project in tree
    • show user tools to edit project

  2. server-side, on new projectDef doc in messageDb:

    • create new project db
    • name it project_userName_projectName (replace @ with _at_ and . with _p_)
    • add user to members in security doc (ensure only members can access it) (do not do this momentarily due to PouchDB issue)
    • delete projectDef doc in messageDb

edit

edit project's user

  1. app-side:

    • user can add users by email
    • app adds user to projectDef.users

  2. server-side, on edit projectDef doc in project db

    • if one of projectDef.users was added:

      • add project name in userDoc's project list
      • add user as member to project db's security doc

    • if one of projectDef.users was removed:

      • remove project name from userDoc's project list
      • remove user from project db's members in security doc

edit project's tables

  1. app-side:

    • user can add table

      • app creates new collection

    • user can remove table

      • app deletes collection

  2. server-side:

    • no action needed

edit other project values

  1. app-side:

    • add projectDef.billing
    • check billing validity?

  2. server-side:

    • no action needed

delete

  1. server-side

    • on delete projectDef doc in project, i.e. db
    • loop projectDef.users
    • remove project name from userDoc's project list
    • delete project db

  2. app-side

    • TODO: need to clear data from rxdb/pouch?
